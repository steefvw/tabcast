# Security Policy

## Supported Versions

This repository is small and currently maintained on a best-effort basis. Security fixes, when available, are expected to land on the latest commit on the default branch rather than on older release lines.

## Reporting a Vulnerability

If you discover a security issue, please avoid posting full exploit details in a public issue.

If private vulnerability reporting is enabled for the published repository, use that channel. Otherwise, contact the maintainer privately through the contact information published on the repository profile or project page before opening a public issue.

When reporting, include:

- A description of the issue and affected component
- Reproduction steps or a proof of concept
- Expected impact
- Any suggested mitigation

Please note that this project is intended for personal or small-scale use and does not currently claim to meet any formal security certification or hardening standard.
